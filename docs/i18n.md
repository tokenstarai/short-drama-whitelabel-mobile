# Mobile i18n

## Locales

The template includes ARB files for:

- `zh`
- `en`
- `th`
- `id`
- `vi`
- `ms`
- `fil`

## Rules

- Buttons, playback errors, legal links, settings, and unlock copy must use localization keys.
- Tenant `defaultLocale` drives the first post-bootstrap runtime language.
- Tenant `supportedLocales` controls the language picker at runtime, but translation files remain in the template.
- The app starts in English when that locale is bundled, then re-renders after `/config` returns the tenant default.
- Server errors must be mapped to App-facing copy before display.

## Runtime Switching

`AppRuntime.strings` is the source for user-facing copy in shell, wallet,
unlock, playback, and account surfaces. `AppRuntime.setLocale(locale)` changes
the active language only when the requested locale is allowed by the current
tenant config or bundled flavor config. The account screen exposes this through
the language picker.

## Error Copy Contract

| App error code | User copy key |
| --- | --- |
| `APP_INSUFFICIENT_BALANCE` | `insufficientBalance` |
| `APP_DRAMA_NOT_AVAILABLE` | `notAvailable` |
| `APP_EPISODE_NOT_READY` | `episodeNotReady` |
| `APP_PLAYBACK_TOKEN_FAILED` | `playbackTokenFailed` |
| `APP_UPSTREAM_UNAVAILABLE` | `serviceMaintenance` |

## Generation

The MVP keeps using `AppStrings` so the open-source template has no generated
localization build step. The ARB files remain the source checklist for future
`gen-l10n` migration.
